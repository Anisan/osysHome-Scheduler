# Руководство пользователя - Scheduler

![Scheduler Icon](../static/Scheduler.png)

Модуль **Scheduler** автоматически запускает Python-код по расписанию. Его используют для регулярных задач по cron-выражению и для одноразовых задач, которые должны выполниться в конкретную дату и время.

> [!IMPORTANT]
> Создавать и изменять задачи планировщика стоит только доверенным администраторам. Каждая задача может выполнять произвольный Python-код внутри системы.

## Назначение модуля

Через `Scheduler` администратор может:

- создавать задачи по расписанию;
- задавать cron-расписание для повторяющегося запуска;
- настраивать одноразовый запуск по дате и времени;
- указывать дату истечения задачи;
- включать и отключать задачу без удаления;
- отслеживать историю выполнения и состояние пула потоков.

## Где находится модуль

Откройте раздел **Администрирование** и выберите **Scheduler**.

## Основные вкладки

В модуле есть две основные вкладки:

- **Tasks**: список настроенных задач;
- **Monitoring**: статистика выполнения, недавняя история и активные задания в пуле потоков.

## Что видно на странице Tasks

В основной таблице обычно отображаются:

- ID задачи;
- имя задачи;
- фрагмент кода;
- время запуска;
- дата истечения;
- время последнего запуска;
- cron-выражение;
- статус активности;
- кнопки действий.

В верхней части страницы также доступны:

- **Refresh** для обновления списка;
- **Add** для создания задачи;
- **Search** для фильтрации по значениям в таблице.

## Основные операции

### Создать задачу

1. Нажмите **Add**.
2. Укажите **Name**.
3. Введите Python-код в поле **Code**.
4. Заполните **Cron** для повторяющейся задачи или **Runtime** для одноразовой.
5. При необходимости задайте **Expire**.
6. Оставьте **Active** включённым, если задача должна начать работать сразу.
7. Нажмите **Submit**.

> [!TIP]
> На практике удобнее использовать для одной задачи либо `Cron`, либо `Runtime`. Так назначение задачи проще понять и сопровождать.

### Изменить задачу

1. Найдите задачу в списке.
2. Нажмите **Edit**.
3. Обновите расписание, код, дату истечения или флаг активности.
4. Нажмите **Submit**.

Чаще всего здесь меняют:

- cron-выражение;
- дату одноразового запуска;
- Python-код;
- дату истечения;
- состояние активности задачи.

### Включить или отключить задачу

1. Откройте список **Tasks**.
2. Для активной задачи используйте кнопку паузы, чтобы отключить её.
3. Для неактивной задачи используйте кнопку запуска, чтобы включить её снова.

Отключённые задачи остаются в списке, но не выполняются, пока вы не включите их обратно.

### Удалить задачу

1. Нажмите кнопку **Delete** в строке нужной задачи.
2. Подтвердите удаление.

> [!WARNING]
> Удаление стирает конфигурацию задачи. Если задача может понадобиться позже, безопаснее сначала отключить её.

## Поля задачи

### Name

Понятное имя задачи, которое показывается в списке, мониторинге и истории выполнения.

### Code

Python-код, который планировщик выполняет при запуске задачи.

> [!CAUTION]
> Перед сохранением внимательно проверяйте код. Ошибки в нём могут вызывать повторяющиеся сбои или влиять на работу системы.

### Cron

Cron-выражение для повторяющегося запуска.

Формат:

`минута час день месяц день_недели`

Планировщик также поддерживает интервалы с точностью до секунд. В этом случае секунды указываются шестым параметром.

Расширенный формат:

`минута час день месяц день_недели секунда`

Примеры:

- `*/5 * * * *` - каждые 5 минут
- `0 7 * * *` - каждый день в 07:00
- `* * * * * */10` - каждые 10 секунд

### Runtime

Дата и время одноразового запуска.

Это удобно, когда задачу нужно выполнить только один раз, например для отложенного действия или обслуживающей операции.

### Expire

Дата и время, после которых задача больше не должна считаться актуальной.

Особенно полезно для временных и одноразовых задач.

### Active

Текущее состояние задачи:

- включённые задачи могут выполняться;
- отключённые остаются в системе, но не запускаются.

## Страница Monitoring

Вкладка **Monitoring** помогает отслеживать работу планировщика в реальном времени. Обычно на ней видны:

- статистика пула потоков;
- статистика времени выполнения;
- график истории времени выполнения;
- текущие активные задачи;
- недавняя история задач с длительностью и статусом успешности.

Эта страница полезна, когда нужно понять, нормально ли выполняются задачи и не перегружен ли пул исполнителей.

## Типовые сценарии

### Настроить регулярную задачу

1. Создайте задачу.
2. Добавьте Python-код.
3. Укажите cron-выражение, например `0 * * * *`.
4. Сохраните задачу с включённым **Active**.
5. После ожидаемого времени запуска проверьте вкладку **Monitoring**.

### Настроить одноразовую задачу

1. Создайте задачу.
2. Заполните **Name** и **Code**.
3. Оставьте **Cron** пустым.
4. Укажите **Runtime** на нужные дату и время.
5. При необходимости задайте **Expire**.
6. Сохраните задачу.

### Приостановить проблемную задачу

1. Откройте список задач.
2. Нажмите кнопку паузы у нужной задачи.
3. Проверьте код и расписание.
4. Включайте задачу обратно только после устранения причины проблемы.

### Проверить сбои

1. Откройте **Monitoring**.
2. Просмотрите недавнюю историю выполнения.
3. Проверьте длительность и статус ошибки.
4. Откройте саму задачу и проверьте её код и расписание.

## Чек-лист администратора

- [ ] У каждой задачи понятное и уникальное имя.
- [ ] Python-код проверен перед активацией.
- [ ] Для задачи выбран правильный способ запуска: `Cron` или `Runtime`.
- [ ] Для временных задач задана дата истечения, если это уместно.
- [ ] Неактуальные или аварийные задачи отключены либо удалены.
